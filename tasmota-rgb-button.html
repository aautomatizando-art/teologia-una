Botão RGBcom Tasmota

Aprenda a configurar um botão físico para controlar cores, brilho e cenas de uma lâmpada RGB — via Console, Rules e comandos MQTT.

fundamentos

O que é o Tasmota?

// firmware open-source

Firmware alternativo para chips ESP8266 e ESP32. Substitui o firmware proprietário de lâmpadas, tomadas e módulos Wi-Fi.

// controle local

Funciona 100% na rede local, sem depender de nuvem. Compatível com Home Assistant, MQTT, e automações via Rules internas.

// hardware suportado

Sonoff, Shelly (ESP), lâmpadas Tuya, Teckin, Gosund, módulos com ESP8266/ESP32 e outros compatíveis com flash.

Cenário deste guia: um dispositivo Tasmota com GPIO de botão físico e saídas PWM para controlar canais R, G, B de uma lâmpada (ou tira de LED RGB).

hardware

O que você precisa

  • 1

    Dispositivo com Tasmota instalado

    ESP8266 / ESP32 com firmware Tasmota. Pode ser lâmpada bulb RGB (ex: Shelly Bulb), módulo LED ou montagem própria com driver PWM.

  • 2

    Botão físico conectado a um GPIO

    Normalmente um botão momentâneo (NO — normalmente aberto) entre o GPIO e o GND. O Tasmota já trata o debounce.

  • 3

    Template/Template configurado

    GPIO do botão definido como Button1 (ou Button2…) no menu Configuration → Configure Module.

  • 4

    Acesso ao Console Tasmota

    Interface web em http://<ip-do-device> → Console. Todos os comandos deste guia são digitados ali.

configuração

Passo a passo: preparar o dispositivo

  • 1

    Acesse o Console → digite os comandos abaixo

    Vá em http://<ip> → Console. Os comandos configuram o módulo para RGB + botão independente.

  • 2

    Desacoplar botão do relay padrão

    SetOption1 1 — agora Button1 não alterna mais o relay automaticamente; será controlado só pelas Rules.

  • 3

    Definir tempo de "segurar" (hold)

    SetOption32 10 → hold detectado após 1 segundo (valor × 100ms). Ajuste conforme preferência.

  • 4

    Proteger contra reset WiFi acidental

    SetOption73 1 → desativa o reset por segurar 4s. Assim o hold fica disponível para suas Rules.

  • 5

    Habilitar multipress

    SetOption13 0 (padrão) — garante que duplo e triplo clique sejam detectados corretamente.

rule 1 — clique simples

1 clique → ligar / desligar

# Apagar qualquer rule antiga no slot 1
Rule1 0

# Definir a rule
Rule1 ON Button1#State=10 DO Power TOGGLE ENDON

# Ativar
Rule1 1

State=10 é o evento de pressão única (single press). Power TOGGLE alterna entre ON e OFF mantendo a última cor definida.

Se quiser ligar sempre na mesma cor ao invés de toggle, substitua por: Power ON e adicione Color #FF4400 na mesma ação separando com ;

# Versão: ligar sempre em laranja ao pressionar
Rule1 ON Button1#State=10 DO Backlog Power ON; Color #FF4400 ENDON
rule 2 — duplo clique

2 cliques → próxima cor

Usamos uma variável Var1 para guardar o índice da cor atual e ciclar entre 5 cores predefinidas.

# Inicializar variável de cor
Var1 1

# Rule2: ciclar cores no duplo clique
Rule2 ON Button1#State=11 DO
Backlog Add Var1 1;
IF (%var1%>5) DO Var1 1 ENDIF;
Color%var1%
ENDON

Rule2 1

// Color1

Vermelho — #FF0000

// Color2

Verde — #00FF00

// Color3

Azul — #0000FF

Alternativa simples: use Color%var1% onde os números 1–25 são as cores predefinidas do Tasmota (listadas em Commands → Light na documentação).

rule 3 — segurar botão

Segurar → modo branco / cena

# Hold: ativa luz branca quente (relaxar)
Rule3 ON Button1#State=3 DO Backlog Power ON; Color #FFFFFF; Dimmer 40; CT 400 ENDON

Rule3 1

// State=3 → hold detectado

  • Ativa ao segurar o botão pelo tempo de SetOption32
  • Ideal para cenas fixas (leitura, cinema, relaxar)
  • Combina vários comandos com Backlog

// outras cenas com hold

  • Cinema: Color #200800; Dimmer 10
  • Festa: Scheme 4; Speed 3
  • Foco: Color #FFFFFF; Dimmer 100; CT 153
  • Dormir: Dimmer 5; CT 500

Atenção: se SetOption73 1 não estiver ativo, segurar 4s aciona o modo WiFi e reinicia o dispositivo — sempre configure antes.

configuração completa

Rule completa em 3 slots

# ═══════════ SETUP INICIAL ═══════════
SetOption1 1 ; desacopla botão do relay
SetOption32 10 ; hold = 1 segundo
SetOption73 1 ; desativa reset WiFi por hold
Var1 1 ; índice de cor inicial

# ═══════════ RULE 1: clique simples → toggle ═══════════
Rule1 ON Button1#State=10 DO Power TOGGLE ENDON
Rule1 1

# ═══════════ RULE 2: duplo clique → próxima cor ═══════════
Rule2 ON Button1#State=11 DO Backlog Add Var1 1; IF (%var1%>6) DO Var1 1 ENDIF; Color%var1% ENDON
Rule2 1

# ═══════════ RULE 3: hold → cena branco quente ═══════════
Rule3 ON Button1#State=3 DO Backlog Power ON; Color #FFF0C0; Dimmer 40; CT 400 ENDON
Rule3 1
controle de brilho

Controlar brilho com botão

Estratégia: hold alterna entre aumentar e diminuir o brilho em passos enquanto o botão fica pressionado.

# Var2 controla direção: 1=subindo, 0=descendo
Var2 1

# Hold: incrementa ou decrementa brilho
Rule3 ON Button1#State=3 DO
Backlog IF (%var2%==1) DO Add Dimmer 10 ENDIF;
IF (%var2%==0) DO Sub Dimmer 10 ENDIF;
IF (%Dimmer%>=100) DO Var2 0 ENDIF;
IF (%Dimmer%<=10) DO Var2 1 ENDIF
ENDON
Rule3 1

Combine Rule1 (toggle) + Rule2 (cor) + Rule3 (brilho) para ter controle completo da lâmpada com um único botão físico.

mqtt · home assistant

Integração com MQTT

// tópicos publicados pelo Tasmota

  • cmnd/LampadaRGB/Power → ON / OFF
  • cmnd/LampadaRGB/Color → valor hex
  • cmnd/LampadaRGB/Dimmer → 0–100
  • stat/LampadaRGB/RESULT → resposta
  • tele/LampadaRGB/STATE → estado periódico

// exemplo Home Assistant (configuration.yaml)

light:
  - platform: mqtt
    name: "Lâmpada RGB"
    command_topic: "cmnd/LampadaRGB/Power"
    rgb_command_topic: "cmnd/LampadaRGB/Color"
    brightness_command_topic: "cmnd/LampadaRGB/Dimmer"
    state_topic: "tele/LampadaRGB/STATE"

Com Tasmota + MQTT, o botão físico e o Home Assistant coexistem: o dispositivo publica seu estado a cada ação, mantendo tudo sincronizado.

resumo

Você tem um botão completo

// 1 clique

Toggle ON/OFF — mantém cor e brilho da última vez que a lâmpada estava ligada.

// 2 cliques

Avança para a próxima cor na sequência configurada (vermelho → verde → azul → …).

// segurar

Ativa cena pré-definida (branco quente, cinema, modo leitura) ou controla brilho progressivamente.

Próximos passos: explore Backlog, IF/ELSEIF/ELSE nas rules e os eventos MQTT para criar automações ainda mais avançadas no Home Assistant ou Node-RED.
